编程范式

Programming Paradigms

主流编程范式范式：描述截然不同的概念或思维模式。

  • 命令式 imperative programming
  • 函数式 functional programming
  • 逻辑式 logic programming

面向对象(object-oriented)正交于以上范式，即每一个范式都能使用面向对象思想。

命令式编程

Imperative programming

命令式编程方式：

  • 修改可变变量
  • 使用赋值
  • 使用控制结构，如if-else、loops、break/continue etc

可以理解为冯·诺依曼计算机的指令序列

命令式编程和计算机

命令式编程是对计算机指令的建模：

  • 可变变量 ≈ 内存单元
  • 变量使用 ≈ load指令
  • 变量赋值 ≈ store指令
  • 控制结构 ≈ 跳转

命令式编程和计算机

难题是"扩展"：怎样避免一个指令一个指令地构建程序。

John Backus, Can Programming Be Liberated from the von. Neumann Style?, Turing Award Lecture 1978.

扩展语言

纯命令式编程会存在 “冯.诺依曼”瓶颈：

逐个指令地构建程序
One tends to conceptualize data structures word-by-word

我们需要其他的技术来定义高级抽象概念，如集合、多项式、几何形状、字符串、文档

理想状态：发展出集合、多项式、几何形状等抽象概念的理论

扩展语言：理论

理论由以下部分组成：

  • 一个或多个数据类型
  • 在这些类型上的操作
  • 描述值和操作之间的定律

扩展语言

解决方法：

  • 集中精力为操作定义理论
  • 最小化可变状态
  • 函数化操作，使之能够组合扩展

函数式编程

Functional Programming

  • 狭义地讲，函数式编程是不使用可变变量、赋值、循环及其他命令式控制结构的编程方式
  • 广义地讲，函数式意味着聚焦在函数上
  • 特别的，函数可以被看做值来进行生产、消费和组合

函数式语言

  • 狭义地讲，函数式语言是不使用可变变量、赋值、循环及其他命令式控制结构的编程语言
  • 广义地讲，函数式语言意味着能以优雅的方式写出聚焦在函数上的代码
  • 特别地，函数在函数式语言中是一等公民。可以被作为生产、消费和组合的值
    • 它们可以在任何地方被定义，包括在其他函数中
    • 它们可以像其他值一样，被作为参数或返回值
    • 它们可以像其它值一样，有一系列操作来组合函数

Higher-Order Functions

函数式语言视函数为一等公民

所以函数可以被当做值一样作为参数，或称为返回值

这种机制提供了灵活地组织程序的方式

使用其它函数为参数或返回值的，称之为高阶函数

Example

对[a b]区间求和


if (a > b) 0 else a + sumInts(a + 1, b)

对[a b]区间求立方和


def cube(x: Int): Int = x * x * x
if (a > b) 0 else cube(a) + sumInts(a + 1, b)

对[a b]区间求factorial和


if (a > b) 0 else factorial(a) + sumInts(a + 1, b)

Example

他们其实可以抽象为一种模式：

$\sum_{n=a}^b f(n)$

我们可以用程序抽象出这种模式吗？

Summing with Higher-Order Functions

答案是可以：


def sum(f: Int => Int, a: Int, b: Int): Int =
	if (a > b) 0
	else f(a) + sum(f, a + 1, b)
	
def sumInts(a: Int, b: Int) = sum(id, a, b)
def sumCubes(a: Int, b: Int) = sum(cube, a, b)
def sumFactorials(a: Int, b: Int) = sum(fact, a, b)

Function Types

The type A => B is the type of a function that takes an argument of type A and returns a result of type B.

所以Int => Int 表示输入为Int，输出为Int的函数

Anonymous Functions

用匿名函数简写


def sumInts(a: Int, b: Int) = sum(x => x, a, b)
def sumCubes(a: Int, b: Int) = sum(x => x * x * x, a, b)
def sumFactorials(a: Int, b: Int) = sum(fact, a, b)

注意到参数a、b不变的传给了sum，能不能去掉这些参数，进一步简化呢？

Functions Returning Functions

首先重写函数


def sum(f: Int => Int): (Int, Int) => Int = {
  def sumF(a: Int, b: Int): Int =
    if (a > b) 0
    else f(a) + sumF(a + 1, b)
  sumF
}

sum 现在的返回值是一个函数

The returned function sumF applies the given function parameter f and sums the results.

Stepwise Applications

我们现在可以这样简写函数：


def sumInts = sum(x => x)
def sumCubes = sum(x => x * x * x)
def sumFactorials = sum(fact)

sumCubes(1, 10) + sumFactorials(10, 20)

Multiple Parameter Lists

由于返回函数的函数，这样的写法非常常见，所以scala提供了一种多参数列表的方式，来表达相同的语法.

例如，前面的函数可以这样重写：


def sum(f: Int => Int)(a: Int, b: Int): Int =
	if (a > b) 0 else f(a) + sum(f)(a + 1, b)

集合上的函数式 java

将一些字符串转为数字，并且取大于1的数，求加和


    String[] strs = new String[]{"1", "2", "3"};
    List stringList = Arrays.asList(strs);

    int total = 0;
    for (String str : stringList) {
      int v = Integer.parseInt(str);
      if (v > 1) {
        total += v;
      }
    }
		
    System.out.println(total);
    assert (total == 5);

集合上的函数式 scala

将一些字符串转为数字，并且取大于1的数，求加和


val stringList = Seq("1", "2", "3")
val intList = stringList map { string => string.toInt }
val bigIntList = intList filter { int => int > 1 }
val total = bigIntList reduce { (a, b) => a + b }
total should be (5)

集合上的函数式 scala one-liner

将一些字符串转为数字，并且取大于1的数，求加和


val stringList = Seq("1", "2", "3")

val total = stringList
  .map(_.toInt)
  .filter(_ > 1)
  .reduce(_ + _)

total should be (5)
